# Requirements

本目录用于记录仓库中的非 bug 需求，不记录 bug。

## 目录用途

- `LEDGER.md`：需求总台账，记录每次收到的需求、分类、状态和归档位置。
- `REQ-*.md`：单条需求的详细记录，适用于需要保留背景、目标、范围、决策和验收标准的需求。

## 分类规则

- 功能需求：
  - 新增能力、接口、页面、流程、配置或规则。
  - 进入 `LEDGER.md`，必要时新增 `REQ-*.md`。
- 功能优化：
  - 对已有能力做增强、补强、限制收紧、流程调整或策略升级。
  - 进入 `LEDGER.md`，必要时新增 `REQ-*.md`。
- 用户体验优化：
  - 对交互、视觉、提示、默认值、操作路径和理解成本的优化。
  - 进入 `LEDGER.md`，必要时新增 `REQ-*.md`。

## 记录规则

1. 每次收到需求，先写入 `LEDGER.md`。
2. 需求复杂、持续时间长、涉及范围广时，再新增独立 `REQ-*.md`。
3. 如果一个请求同时包含 bug 和优化项，必须拆分为多条记录。
4. bug 不写入 `Requirements/`，统一记录在 `docs/BUG_TRACKER.md`。
5. 需求完成后，要回填状态、落地文档和验证结果。

## 状态建议

- `待评估`
- `进行中`
- `已完成`
- `已搁置`
- `已取消`
