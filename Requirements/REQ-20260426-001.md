## 需求标题

将 Notus 按懒猫微服规范移植为单容器应用，并补齐 `lpk` 打包与发布所需文件。

## 背景

仓库内已有懒猫相关脚本和清单，但结构和字段与当前规范不完全一致，缺少 `package.yml`、正式图标和标准化 `lzc-build.yml`，现有打包脚本也还是手工压缩归档，不能直接作为规范的 `lpk` 打包入口。

## 目标

1. 采用 `v1` 单容器方案完成懒猫微服移植。
2. 补齐商店发布所需的元数据、图标和打包文件结构。
3. 固化 Linux amd64 的 Next.js 构建流程，避免原生依赖架构不匹配。
4. 保持业务 HTTP 接口和应用功能不变，只调整部署与打包链路。

## 范围

- 新增或调整 `package.yml`、`lzc-manifest.yml`、`lzc-build.yml`、`lzc/run.sh`、`lzc/build-package.sh`
- 生成 `lzc-icon.png`
- 按需更新 `README.md` 与 `docs/PROGRESS.md`
- 不修改业务 API、页面路由和产品交互

## 预期验收

1. 打包文件齐全，字段满足懒猫规范。
2. `build-package.sh` 只负责准备 `lzc-dist/`，不再自行产出 `.lpk`。
3. 运行时目录只使用 `/lzcapp/var`、`/lzcapp/cache`，不写入 `/lzcapp/pkg/content`。
4. 能通过脚本语法检查、Next.js 构建检查，并具备继续执行 `lzc-cli project build` 的条件。

## 落地结果

1. 新增 `package.yml`、`LICENSE`、`notus/.npmrc` 和 `lzc-icon.png`。
2. 按懒猫工具实际要求收窄 `lzc-manifest.yml`，仅保留运行时配置；静态元数据统一放入 `package.yml`。
3. 重写 `lzc-build.yml` 和 `lzc/build-package.sh`，由 `build-package.sh` 负责准备 `lzc-dist/`，由 `lzc-cli project build` 负责生成 `.lpk`。
4. 重写 `lzc/run.sh`，只创建运行时目录并启动 Next.js standalone 服务。
5. 更新 `README.md` 与 `docs/PROGRESS.md`，同步新的懒猫打包说明与当前验证状态。

## 验证

1. `sh -n lzc/build-package.sh` 通过。
2. `sh -n lzc/run.sh` 通过。
3. `cd notus && npm run lint` 通过。
4. `cd notus && npm run build` 通过。
5. `LZC_BUILD_IN_CONTAINER=1 lzc-cli project build -o notus-local-test.lpk` 通过，并生成本地测试包。
6. `lzc-dist/` 中已确认存在 `notus/server.js`、`notus/.next/static`、`notus/public` 和 `lzc/run.sh`。
7. 已通过 `lzc-cli appstore copy-image node:20-bookworm-slim` 获取懒猫官方镜像地址 `registry.lazycat.cloud/u30387910/library/node:549f023f95a10c59`，并回写到运行时与构建时默认镜像配置。
