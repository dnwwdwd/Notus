# REQ-20260427-003

## 基本信息

- 日期：2026-04-27
- 分类：用户体验优化
- 状态：已完成
- 来源：用户请求

## 需求标题

优化引导页、文件树、导入弹窗与 AI 锁态交互细节。

## 背景

用户反馈当前版本仍有一批非 bug 的交互与视觉细节没有收口，包括：

1. 初始化引导与设置页宽度偏窄，长模型配置文本显示不完整。
2. 文件树索引状态图标语义不清，缺少 tooltip，搜索态反馈不足。
3. 导入 Markdown 弹窗的上传方式、按钮启用条件与结果态文案不够顺手。
4. 知识库与创作页在模型未配置时缺少明确锁态，创作页仍暴露不合理按钮。
5. 第一版暂不交付 AI 输入框附件能力，需要明确下沉到 2.0 实现文档。

## 预期落地

1. 调整 `/setup` 与 `/settings/*` 的内容宽度和提示文案。
2. 完善侧边文件树的状态图标、tooltip、搜索聚焦、高亮和关闭搜索行为。
3. 重构 Markdown 导入弹窗，支持拖拽/文件/目录上传，收紧按钮状态与结果态文案。
4. 为知识库与创作页补齐统一的 AI 未就绪锁态，并移除创作页不必要的显式按钮。
5. 新建或更新 2.0 实现文档，记录附件与扩展能力延期到后续版本。

## 验证

1. 已执行 `npm run lint`，通过，无 ESLint 报错。
2. 已执行 `npm run build`，通过，`/setup`、`/files`、`/knowledge`、`/canvas`、`/settings/[section]` 均成功产出。
3. 页面级烟测仍建议在真实浏览器中继续回归引导页、文件树、导入弹窗、知识库页与创作页关键交互。
