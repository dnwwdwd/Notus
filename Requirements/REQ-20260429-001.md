## 需求标题

升级知识库问答、创作工作台与全局工作区状态。

## 背景

用户确认核心链路已经可用后，进一步提出了这一轮按业务功能分类的升级要求：

1. 文件页、知识库页、创作页之间缺少统一的“当前文档”状态，切页后仍需要二次选择或重新打开文档。
2. 知识库问答虽然能检索来源，但回答更像“告诉你去哪看”，而不是结合具体召回内容做总结整理。
3. 来源卡片跳转仍停留在文件页逻辑，不符合知识库场景下“边问边看边定位原文”的使用方式。
4. 创作页需要把“事实上下文”和“风格来源”拆开处理，并让当前文档在生成大纲和 AI 改写时真正参与上下文。
5. 文章分块过细，缺少按标题层级和语义组织的稳定切块策略。
6. 侧边栏收缩、文件树展开等桌面工作区状态需要全局持久化；本版本不做移动端适配。

这些内容属于现有能力的业务逻辑升级，按“功能优化”归档。

## 预期落地

1. 建立全局工作区状态，统一维护当前页面、当前文档、文件树展开状态、侧边栏折叠状态与待消费的来源定位目标。
2. 在知识库页补齐更完整的问答链路：当前文档优先、章节聚合、证据充分性判断、来源点击后页内编辑器定位与高亮。
3. 在创作页让事实上下文与风格来源分别进入生成链路，并让大纲生成与 Agent 改写同时读取当前文档、后台事实补充结果和风格样本。
4. 将 Markdown 文章分块升级为“标题层级优先，语义分块回退”的结构化算法，并输出更丰富的块元信息。
5. 补写一份按业务功能分类的升级文档，并同步清理 `PROGRESS`、PDD、PRD、UI Guide、TODO 中与当前实现冲突的旧口径。
