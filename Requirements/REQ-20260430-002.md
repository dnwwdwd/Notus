## 需求标题

为知识库页与创作页引入模型上下文预算感知、自动 compact 与上下文窗口配置持久化。

## 背景

用户明确要求系统能够识别不同模型的上下文窗口，在接近上限时自动 compact，而不是等模型超限后再被动报错。当前实现虽然已经有“最近 12 条历史 + 当前问题 + RAG”的基础链路，但缺少：

1. 每条 LLM 配置的上下文预算持久化。
2. 对知识库页和创作页的统一 token 预算控制。
3. 接近上限时的自动裁剪和压缩策略。
4. usage / budget / compacted 的运行态回传。

因此按“功能优化”收口。

## 预期落地

1. `llm_configs` 持久化上下文窗口和默认输出上限，并支持已知模型自动填充。
2. 所有主要 LLM 调用链路统一接入预算层，支持 usage 采集、超限识别与最多一次自动重试。
3. 知识库页在保留证据优先原则下，自动压缩历史、sections 和 chunks。
4. 创作页改为“相关块包 + 历史摘要 + 事实/风格裁剪”入模，不再默认整篇文章全量入模。
5. 业务文档同步更新为新口径。

## 验证

1. 已新增统一预算与 compact 工具层，打通知识库问答、创作 Agent、大纲生成、块级改写和连接测试。
2. 已为 LLM 配置持久化 `context_window_tokens` 与 `max_output_tokens`，并在设置页展示自动识别结果。
3. 已更新知识库页与创作页的 chat 业务文档，补充预算、usage 和 compact 逻辑。
4. 已执行 `npm run lint`，通过。
