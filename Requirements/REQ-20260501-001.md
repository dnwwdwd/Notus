# REQ-20260501-001

## 基本信息

- 日期：2026-05-01
- 分类：用户体验优化
- 状态：已完成
- 来源：用户请求

## 需求标题

优化 Header、侧边栏收起态与 AI 输入框响应式。

## 背景

当前 Header 在应用宽度变窄时仍保留较多文字与固定宽度控件，容易造成拥挤；侧边栏收起后宽度仍偏大；创作页 AI 输入框在右侧面板较窄时会被模型选择器和发送按钮挤压，出现文字竖向排版，同时空态输入框高度明显高于知识库页。

这些问题不改变功能边界，但会影响窄窗口和分栏场景下的可用性，因此按“用户体验优化”归档。

## 预期落地

1. Header 在窄宽度下逐级简化为图标操作，并为图标按钮补充 tooltip。
2. Header 的导航、保存、搜索、设置和文件名区域在不同宽度下保持不换行、不遮挡。
3. 侧边栏收起宽度缩小到接近单个展开按钮宽度。
4. 创作页 AI 输入框高度与知识库页保持接近，窄宽度下输入框不被挤成竖向排版。
5. 不改变现有导航、保存、搜索、AI 发送、模型选择和侧边栏切换行为。

## 验证

1. 已更新 Header 响应式降级、侧边栏收起宽度和 AI 输入框窄宽度换行规则。
2. `npm run lint` 通过。
3. `npm run build` 通过。
4. 已用本地开发服务检查 `/files`、`/knowledge`、`/settings/model` 在 1200px、900px、680px 宽度下无横向溢出；侧边栏收起后宽度为 40px。
5. 当前本地环境下 `/canvas` 未渲染 AI 输入框，创作页输入框改动已通过组件代码与构建验证。
