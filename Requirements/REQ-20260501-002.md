## 需求标题

固定 AI 输入框模型选择器位置与上拉行为。

## 背景

知识库页和创作页共用的 AI 输入框中，模型选择器会跟随输入框宽度变化而变化，且弹出方向是下拉。用户希望模型选择器宽度固定，视觉上仍保持 dropdown 组件效果，但菜单从输入框上方弹出，并固定放在输入框右下角、发送按钮左侧。

这不改变模型选择和发送行为，只调整输入框区域的布局和交互呈现，因此按“用户体验优化”归档。

## 预期落地

1. 模型选择器宽度固定，不随输入框宽度变化。
2. 模型选择器固定在 AI 输入框右下角，位于发送/停止按钮左侧。
3. 模型菜单向上弹出，避免遮挡输入框下方区域。
4. 维持现有 `DropdownSelect` 组件、模型选择逻辑和发送逻辑不变。

## 验证

1. [notus/components/ChatArea/InputBar.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/ChatArea/InputBar.js) 已将模型选择器固定为 150px，并放在输入框右下角、发送按钮左侧。
2. 模型选择器继续使用 `DropdownSelect`，菜单弹出方向改为 `placement="top"`。
3. `npm run lint`、`npm run build` 通过。
4. 已用浏览器检查 `/knowledge`，模型选择器宽度为 150px，发送按钮位于右侧，页面无 hydration 报错；当前本地无可用模型配置，菜单为禁用态。
