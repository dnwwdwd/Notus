# REQ-20260501-006 桌面端并存改造与平台中间层

## 背景

用户要求在不新开分支的前提下，把 Notus 从原先偏向懒猫运行时的形态调整为 Web 与 Electron 桌面端并存，同时把不同平台的配置、路径、能力和专用 API 统一收口到中间层，避免业务代码再次散写平台判断。

## 目标

1. 新增平台中间层，统一解析运行目标、数据目录、能力开关和平台资料。
2. 新增 Electron 桌面壳，复用现有 Next.js standalone 服务，不复制业务后端。
3. 桌面端采用应用内托管工作区，笔记、附件、数据库、日志和浏览器会话统一落到应用专属目录。
4. 懒猫相关打包文件和迁移文档不再纳入 Git 管理，但代码层继续保留兼容。
5. 更新项目文档和仓库脚本，形成新的主线口径。

## 实施范围

- 新增 `notus/lib/platform/`、`notus/utils/platformClient.js`、`notus/contexts/PlatformContext.js`
- 更新 `/api/health`、`/api/settings`、`/api/setup/status`
- 更新 `/setup` 与设置页的存储、关于文案和桌面能力入口
- 新增 `desktop/` 目录、根目录 `package.json` 与桌面构建脚本
- 更新 `.gitignore` 与关键文档

## 验收标准

1. Web 与桌面端共用一套 Next.js 业务代码和 API。
2. 桌面端可通过本地对话框导入 Markdown，并把数据写入托管工作区。
3. 平台信息可通过接口和前端上下文读取，不再依赖页面内硬编码路径或平台判断。
4. 设置页可显示当前运行平台、数据根目录，并在桌面端提供打开数据目录、清除本机数据并退出入口。
5. 懒猫相关打包文件和迁移文档已移出 Git 管理范围。

## 结果

- 已完成平台中间层、Electron 桌面壳、桌面导入桥接、设置页桌面能力入口和根目录脚本接入。
- 已更新 `.gitignore`、README、AGENTS、CLAUDE、PDD、PRD、`PROGRESS` 等关键文档口径。
- 桌面端安装、卸载和清理流程仍需在真实 Windows / macOS 环境下继续验证。
