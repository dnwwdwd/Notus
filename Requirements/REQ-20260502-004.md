# REQ-20260502-004 同步知识库实现到产品文档并强化 PDD/PRD 维护规则

## 背景

知识库单索引问答链路已经连续完成多轮升级，但 `docs/Notus_PDD.md` 和 `docs/Notus_PRD.md` 里还没有完整反映最新实现，尤其缺少以下内容：

1. 单索引约束和成本护栏
2. 查询理解、澄清追问、条件改写和条件重排
3. 回答模式与保守回答边界
4. `messages.meta`、SSE 事件和检索链路的最新实现

同时，仓库规则里此前只要求“按需更新”产品文档，约束不够硬，容易出现功能已经改了但 PDD / PRD 没有同步的情况。

## 目标

1. 将当前知识库实现同步写入 `docs/Notus_PDD.md` 和 `docs/Notus_PRD.md`。
2. 在协作文档中明确规定：只要项目功能实现发生变化，就必须同步更新 PDD 和 PRD。
3. 保持 `AGENTS.md` 与 `CLAUDE.md` 完全一致。

## 结果

- 已在 `docs/Notus_PDD.md` 补齐知识库问答的当前产品口径，包括单索引策略、澄清追问、辅助调用上限、回答模式和知识库页状态。
- 已在 `docs/Notus_PRD.md` 补齐当前技术实现，包括新模块目录、`messages.meta`、知识库检索输出结构、helper 护栏、`/api/chat` SSE 事件和缓存规则。
- 已在 `AGENTS.md` 与 `CLAUDE.md` 中新增硬性规则：凡是功能实现变更，必须同步更新 `docs/Notus_PDD.md` 和 `docs/Notus_PRD.md`，不能等用户额外提醒。
