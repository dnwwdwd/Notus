# REQ-20260502-006 收紧关于页文案并重做 LLM 配置卡片展示

## 分类

- 类型：用户体验优化
- 状态：已完成

## 背景

当前设置页和引导页 Step 1 的 LLM 配置卡片信息偏多，视觉层级也比较散。卡片里同时展示最近测试、上下文窗口和默认输出上限，会把重点从“这是什么配置、连到哪里、能不能直接用”转移开。

另外，关于页中的三段运行环境说明过长，信息密度偏低。

## 本次目标

1. 去掉关于页中过长的运行环境说明。
2. 让设置页和引导页 Step 1 的 LLM 配置卡片只保留高价值信息。
3. 调整卡片布局和视觉层级，让模型配置页更简洁、现代、易扫读。

## 实施结果

### 1. 关于页收短

- 移除了关于页原先的三段运行环境说明。
- 现在只保留品牌、版本号和一句简短定位说明。

### 2. LLM 卡片信息收口

- 设置页和引导页 Step 1 的 LLM 配置卡片不再展示：
  - 最近测试
  - 上下文窗口
  - 默认输出上限
- 卡片保留的信息改为：
  - 配置名称
  - 厂商识别
  - 模型名称
  - Base URL
  - 默认配置状态
  - API Key 是否已保存

### 3. 卡片视觉重做

- 重做了 LLM 配置卡片的头部层级、信息区和底部状态区。
- 默认配置会有更清晰的强调色和状态标识。
- Base URL 和模型名称改为更紧凑的双卡片信息布局。
- 说明性文字改为单行状态区，不再把低频信息堆在主卡片中。
- 引导页 Step 1 的 Embedding 卡片和 LLM 卡片容器也同步调整为更统一的卡片风格。

### 4. 表单说明同步收短

- 新增或编辑 LLM 配置时，不再在弹窗里展示上下文窗口和默认输出上限的说明。
- 测试成功提示也不再展示测试耗时。
