# REQ-20260502-008 升级创作页意图识别为受控增强并加入轻量纠偏

## 分类

- 类型：功能优化
- 状态：已完成

## 背景

创作页虽然已经有结构化提问卡片，但上一版主要解决的是“问不问、怎么续跑”的问题，还没有把意图识别本身收紧到足够稳定的程度。几个明显风险仍然存在：

1. 对“把上面的内容写到文档中”“把刚才生成的内容写进去”这类请求，系统可能没把主意图、来源内容、目标位置和写入方式拆开判断。
2. 高风险修改在预算不足、helper 失败或候选块不稳时，容易退化成不够明确的猜测。
3. 用户纠正过“不是聊天，是改文档”“目标不对”“来源不对”后，系统不会在后续几轮短时间记住这些纠正。
4. 前端虽然能显示提问卡片，但缺少对“系统当前是怎么理解的”的回显，也缺少更轻的纠偏入口。

## 本次目标

1. 将创作规划升级为“候选召回 -> 受控 AI 仲裁 -> 风险校验 -> 交互”的固定链路。
2. 引入主意图候选、来源内容分型、细粒度写入槽位和风险等级，不再只靠旧的 `intent + target_block_ids`。
3. 在不常驻模式切换的前提下，为高风险预览补上决策摘要和轻量纠偏入口。
4. 把提问卡片回答也写入会话内短时纠错记忆，提升后续几轮的识别稳定性。

## 实施结果

### 1. 规划器升级为受控增强链路

- `resolveCanvasRequest()` 现在固定先产出：
  - `intent_candidates`
  - `primary_intent`
  - `target_candidates`
  - `source_candidates`
  - `source_content_type`
  - `target_anchor / position_relation / write_action`
  - `risk_level / decision_path / decision_summary / ai_arbitration_mode`
- 规则层不再直接拍板所有情况，而是先产出候选，再决定是否需要单次 AI 仲裁。
- 高风险场景只在必要时触发一次额外风险核验，不扩张成多轮 helper。

### 2. “写到文档中”链路加入来源分型和冻结

- 来源内容统一分为：
  - `draft_text`
  - `edit_suggestion`
  - `analysis_feedback`
  - `general_chat`
- 只有 `draft_text` 能直接进入“写到文档中”的写入链路。
- 规划器会优先冻结：
  - `source_message_id`
  - `source_content_snapshot`
  - `source_content_digest`
  - `source_content_type`
- 如果来源不合格，不再退化为“拿最近一条聊天消息凑数”。

### 3. 主意图、写入位置和写入方式全部显式化

- 提问卡片新增 `primary_intent` 槽位。
- 回答“继续讨论”或“文章分析”后，系统不会再强行要求补齐 `source_content_ref / target_location / write_mode`。
- 写入位置和动作细化为：
  - `target_anchor`
  - `position_relation`
  - `write_action`
- 卡片回答与续跑计划会统一落到这套结构，不再只靠旧的 `write_mode` 文本值。

### 4. 增加决策摘要和轻量纠偏入口

- 助手在以下场景会回显当前判断摘要：
  - 刚完成结构化澄清后继续生成预览
  - 高风险预览
  - 带纠错记忆继续生成的预览
- 创作页预览消息下方新增一次性轻量纠偏入口：
  - `继续讨论`
  - `直接改文档`
  - `目标不对`
  - `来源不对`
  - `写入方式不对`
- 这些纠偏会以 `user_meta.correction_state` 形式回传后端，不需要用户先切到其他模式。

### 5. 会话内短时纠错记忆补齐

- 规划器现在会读取并短时记住：
  - `wrong_intent`
  - `wrong_target`
  - `wrong_source`
  - `wrong_write_action`
- 纠错来源包括：
  - 普通用户输入
  - 轻量纠偏按钮
  - 提问卡片回答摘要消息的 `meta.correction_state`
- 这层记忆只作用在当前会话和当前文章 hash，不做运行时自学习。

### 6. 前后端与测试同步补齐

- `/api/agent/run` 新增支持：
  - `user_meta`
  - 更完整的 `assistant_meta`
  - interaction payload 中的 `decision_summary / risk_level / ai_arbitration_mode / source_content_type / correction_state`
- `/api/interactions/:id/respond` 现在会把卡片回答写入纠错记忆。
- 前端创作页支持展示：
  - 决策摘要
  - 轻量纠偏按钮
  - 卡片中的“当前理解”说明
- 已补齐并通过：
  - `canvas-request-planner`
  - `canvas-agent`
  - `conversation-interactions`
  - `lint:web`

## 当前边界

1. 本轮仍只优化创作页，不抽共享层到知识库页。
2. 轻量纠偏入口用于修正明显误解，不是常驻模式开关。
3. 继续保持“只生成待应用预览，不自动把内容写回文档”。
