# REQ-20260504-001 设置页与引导页的 Embedding 输入框默认留空

## 基本信息

- 日期：2026-05-04
- 分类：用户体验优化
- 状态：已完成
- 来源：用户提出

## 需求标题

设置页与引导页中的 Embedding Base URL、模型名和 API Key 输入框在初次打开时默认留空。

## 背景

当前设置页和引导页会自动把已保存的 Embedding Base URL 与模型名回填到输入框中。用户这次希望这三个输入框默认保持空白，只在需要修改时自行填写，避免每次进入页面都直接看到旧配置内容。

## 预期落地

1. 设置页中的 Embedding Base URL、模型名、API Key 输入框初次展示时默认留空。
2. 引导页 `/setup` 第 1 步中的 Embedding Base URL、模型名、API Key 输入框初次展示时默认留空。
3. 已保存的 API Key 状态继续通过占位提示“已保存，留空不修改”体现。
4. 已识别的 Embedding 维度和多模态开关状态继续保留原有展示与保存逻辑。

## 验证

1. [notus/components/Settings/SettingsScreen.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Settings/SettingsScreen.js) 已移除 Embedding Base URL 和模型名的默认预填与已保存值回填。
2. [notus/pages/setup.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/pages/setup.js) 已移除引导页 Step 1 的 Embedding Base URL 和模型名默认预填与已保存值回填。
3. 设置页与引导页仍保留 API Key 的“已保存，留空不修改”占位提示。
4. `npm run lint:web` 通过。
