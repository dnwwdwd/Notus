# REQ-20260505-001 明确 LPK 包标识符并声明不支持移动端

## 基本信息

- 日期：2026-05-05
- 分类：功能优化
- 状态：已完成
- 来源：用户提出

## 需求标题

将 `.lpk` 包标识符统一为 `cloud.lazycat.app.notus`，并在懒猫包元数据中显式声明不支持移动端。

## 背景

当前项目已经按桌面工作区形态设计，不提供手机和平板适配。虽然文档里已经说明“不做移动端适配”，但懒猫 `.lpk` 元数据里还没有显式声明 `ios` 和 `android` 为不支持平台。用户这次希望同时确认包标识符使用 `cloud.lazycat.app.notus`，避免后续分发和安装侧继续误判平台能力。

## 预期落地

1. `.lpk` 的包标识符保持为 `cloud.lazycat.app.notus`。
2. 在懒猫 LPK v2 的静态元数据文件 `package.yml` 中新增 `unsupported_platforms`。
3. 显式声明 `ios` 与 `android` 不受支持。
4. 重新生成新的 `.lpk` 产物。

## 落地结果

1. 已确认 [package.yml](/Users/hejiajun/Documents/lzc_projects/Notus/package.yml) 中的包标识符保持为 `cloud.lazycat.app.notus`。
2. 已在 [package.yml](/Users/hejiajun/Documents/lzc_projects/Notus/package.yml) 中新增：
   - `unsupported_platforms:`
   - `ios`
   - `android`
3. 已同步更新 [docs/Notus_PDD.md](/Users/hejiajun/Documents/lzc_projects/Notus/docs/Notus_PDD.md) 与 [docs/Notus_PRD.md](/Users/hejiajun/Documents/lzc_projects/Notus/docs/Notus_PRD.md)，让产品口径、技术文档和实际打包配置一致。

## 验证

1. `npm run dist:lpk` 成功。
2. 新生成的 `.lpk` 产物文件名为 `cloud.lazycat.app.notus-v0.1.2.lpk`。
3. 解包检查后，包内 `package.yml` 包含 `package: cloud.lazycat.app.notus` 与 `unsupported_platforms: [ios, android]` 对应配置。
