# REQ-20260506-001 单文件导出时直接下载 Markdown

## 基本信息

- 日期：2026-05-06
- 分类：用户体验优化
- 状态：已完成
- 来源：用户提出

## 需求标题

导出文件时，如果只选中了一个文件，就直接下载 `.md` 文件，而不是打成压缩包。

## 背景

当前侧边栏导出接口无论选中一个还是多个文件，都会统一生成 zip。对于只导出单篇 Markdown 的常见场景，这会多出一次解压步骤，实际使用上不够直接。

## 预期落地

1. 导出弹窗只选中 1 个文件时，直接下载该文件的原始 Markdown 内容。
2. 下载文件名保持原始文件名，例如 `xxx.md`。
3. 选中多个文件时，继续沿用现有 zip 导出方式。
4. 前端导出入口保持一套交互，不额外拆分按钮。

## 落地结果

1. [notus/pages/api/files/export.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/pages/api/files/export.js) 已支持单文件导出直接返回 `text/markdown; charset=utf-8`。
2. 单文件导出时，下载文件名会使用原始 Markdown 文件名；多文件导出仍返回 zip。
3. [notus/components/Layout/Sidebar.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Layout/Sidebar.js) 继续复用同一套下载流程，无需新增额外入口。
4. 已同步更新 [docs/Notus_PDD.md](/Users/hejiajun/Documents/lzc_projects/Notus/docs/Notus_PDD.md) 与 [docs/Notus_PRD.md](/Users/hejiajun/Documents/lzc_projects/Notus/docs/Notus_PRD.md)。

## 验证

1. `npm run lint:web` 通过。
2. `npm run build:web` 通过。
