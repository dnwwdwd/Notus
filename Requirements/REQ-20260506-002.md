# REQ-20260506-002 支持 KaTeX 数学公式编辑与渲染

## 基本信息

- 日期：2026-05-06
- 分类：功能需求
- 状态：已完成
- 来源：用户提出

## 需求标题

在 Notus 中新增数学公式能力，统一使用 KaTeX，让 Web 端和桌面端都支持 Markdown 公式的编辑、渲染和显示。

## 背景

当前项目已经引入了 `katex` 样式和 `rehype-katex`，但实际链路并不完整。所见即所得编辑器没有接入数学节点，Markdown 展示链路也缺少 `remark-math`，导致 `$...$` 与 `$$...$$` 公式语法无法稳定识别和显示。

## 预期落地

1. 文件页与知识库页的所见即所得编辑器支持 Markdown 数学公式。
2. 数学公式统一使用 KaTeX 渲染，兼顾性能和兼容性。
3. Markdown 预览与流式回答展示链路也要支持公式，避免不同区域口径不一致。
4. Web 与桌面端共用同一套实现，不额外分叉。

## 落地结果

1. [notus/components/Editor/WysiwygEditor.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Editor/WysiwygEditor.js) 已接入官方 `@tiptap/extension-mathematics`，统一处理行内公式和块级公式。
2. 编辑器中的公式统一按 KaTeX 渲染，并启用 `output: 'htmlAndMathml'`、`throwOnError: false`、`strict: 'ignore'`，兼顾性能、可访问性和容错。
3. 已为编辑器补充点击编辑入口，用户点击现有公式即可修改 LaTeX 内容。
4. [notus/components/Editor/MarkdownPreview.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Editor/MarkdownPreview.js) 与 [notus/components/ui/StreamingText.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/ui/StreamingText.js) 已补齐 `remark-math + rehype-katex`，统一支持 `$...$`、`$$...$$`。
5. [notus/styles/globals.css](/Users/hejiajun/Documents/lzc_projects/Notus/notus/styles/globals.css) 已补充公式节点样式，让编辑态在 Web 与桌面端保持一致。

## 验证

1. `node notus/tests/editor-math-support.test.js` 通过。
2. `npm run lint:web` 通过。
3. `npm run build:web` 通过。
