# REQ-20260508-001 统一 AI loading 反馈并补齐编辑器居中与最小尺寸约束

## 基本信息

- 日期：2026-05-08
- 分类：用户体验优化
- 状态：已完成
- 来源：用户提出

## 需求标题

统一知识库页与创作页的 AI loading 反馈样式，并补齐编辑器文本居中能力，以及 Web / Electron 两端的最小宽高约束。

## 背景

当前知识库页和创作页在 AI 回复期间只有输入框区域会出现“停止生成”按钮，正文区域缺少同步反馈，看起来像没有进入生成状态。与此同时，编辑器还缺少文本居中能力；而在窗口过窄时，不论是 Web 端还是桌面端，页面内容都会开始相互挤压，影响使用。

## 预期落地

1. 知识库页与创作页在 AI 生成期间，聊天区本身要有清晰、简洁、与现有配色一致的 loading 反馈。
2. loading 主色需要与发送按钮保持一致，不额外引入突兀的新颜色。
3. 编辑器支持文本内容居中，并且保存后可以继续回显，不是仅当前渲染态生效。
4. Web 端与 Electron 桌面端都要设置最小宽度和最小高度，避免缩放过小时布局堆叠。

1. [notus/components/ChatArea/ChatMessage.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/ChatArea/ChatMessage.js) 已为检索中与 AI 回复中统一接入暖色系点阵 loading，颜色与项目 accent、发送按钮保持一致。

3. [notus/components/Editor/TextAlignCenterExtension.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Editor/TextAlignCenterExtension.js)、[notus/components/Editor/WysiwygEditor.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Editor/WysiwygEditor.js) 与 [notus/components/Editor/EditorToolbar.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Editor/EditorToolbar.js) 已补齐“文本居中”扩展、工具栏入口和样式，使居中内容可以随 Markdown 一起保存与回读。
