# REQ-20260510-001 文本编辑器支持文档内图片预览与左右切换

## 基本信息

- 日期：2026-05-10
- 分类：用户体验优化
- 状态：已完成
- 来源：用户提出

## 需求标题

为文本编辑器中的图片补齐预览能力，支持在同一篇文档内查看大图，并通过左右方向键切换上一张或下一张图片。

## 背景

当前所见即所得编辑器已经支持插入本地图片和外链图片，但编辑态里点击图片只能看到原位缩略图，无法放大查看，也不能在同一文档的多张图片之间快速切换。对于带多张配图的文章，用户需要反复滚动页面或依赖浏览器单独打开图片，效率较低。

## 预期落地

1. 文件页与知识库页复用的所见即所得编辑器都支持图片预览，不额外拆出两套实现。
2. 点击编辑器正文中的任意图片后，弹出预览层并显示当前图片的大图。
3. 预览切换范围限定在当前文档内，支持左右方向键切换上一张或下一张图片。
4. 预览层支持 `Esc` 关闭，并在只有一张图片时自动禁用左右切换。

## 落地结果

1. [notus/components/Editor/WysiwygEditor.js](/Users/hejiajun/Documents/lzc_projects/Notus/notus/components/Editor/WysiwygEditor.js) 已内置轻量图片预览层，点击编辑器正文图片即可打开当前文档范围内的预览。
2. 预览逻辑会按当前文档中的出现顺序收集图片列表，并以点击项作为初始位置，保证重复图片地址也能按文内顺序正确切换。
3. 预览层已支持 `ArrowLeft`、`ArrowRight` 与 `Escape` 键盘操作，同时在预览打开时锁定页面滚动，减少误触带来的干扰。
4. [notus/styles/globals.css](/Users/hejiajun/Documents/lzc_projects/Notus/notus/styles/globals.css) 已补齐暖色系蒙层、左右切换按钮、计数信息和大图容器样式，整体配色与现有编辑器风格保持一致。
5. [docs/Notus_PDD.md](/Users/hejiajun/Documents/lzc_projects/Notus/docs/Notus_PDD.md) 与 [docs/Notus_PRD.md](/Users/hejiajun/Documents/lzc_projects/Notus/docs/Notus_PRD.md) 已同步更新当前产品行为和实现口径。
