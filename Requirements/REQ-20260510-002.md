# REQ-20260510-002 重新定义 Notus 为 Markdown 工作区 AI 协作环境

## 基本信息

- 日期：2026-05-10
- 分类：功能优化
- 状态：已完成
- 来源：用户提出

## 需求标题

将 Notus 的长期产品形态从“知识库问答 + 普通文件改写”重新定义为“本地 Markdown 工作区中的 AI 知识与写作协作环境”。

## 背景

旧产品文档对 Notus 的描述偏向知识库问答、单文件创作和普通改写，这不能准确表达目标产品形态。用户期望 Notus 更接近 Obsidian 与 Claude Code / Codex 的组合：Markdown 文件系统仍由用户直接掌控，AI 则作为工作区协作者，在明确范围内阅读、分析、生成预览并执行经用户确认的修改。

## 目标口径

1. `.md` 文件仍是真相来源，数据库只保存索引、缓存、会话、预览和运行状态。
2. Notus 的核心不只是聊天，也不只是单文件改写，而是面向 Markdown 工作区的 AI 协作环境。
3. Agent 的上下文应来自当前文件、目录、选中文件、检索证据、风格参考、对话历史和用户可见范围。
4. 所有写入 Markdown 的能力都必须先形成可审查结果；单文件使用块级 diff，多文件使用批量预览。
5. 后续能力扩展应围绕工作区工具展开，例如读取文件、搜索工作区、创建笔记、更新 frontmatter、生成多文件预览、整理目录和检查内部链接。
6. 通用知识回答必须与用户笔记依据分开表述，不能把模型自身知识伪装成来自用户工作区的证据。

4. 工作区工具注册表与权限边界。
