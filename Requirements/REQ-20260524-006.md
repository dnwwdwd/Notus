# REQ-20260524-006 新增仓库级 LPK 重打包脚本与使用说明

## 基本信息

- 日期：2026-05-24

## 需求标题

新增一个仓库脚本，用于执行 `.lpk` 打包时先删除旧的 `.lpk`，再生成新的产物，并把这个脚本的用途与使用方式记录到协作规范文档中。

## 背景

仓库里已有 `.lpk` 打包能力，但日常手动执行时，团队仍需要显式记住“先删旧包，再打新包”的约束。把这一步收敛成单独脚本，可以减少重复操作，也能让使用入口更直观。

## 本次目标

1. 新增一个可直接执行的仓库脚本，统一完成旧 `.lpk` 删除和新包打包。
2. 为脚本补一个 `npm` 快捷入口，方便团队直接使用。
3. 在 `AGENTS.md` 和 `CLAUDE.md` 中记录脚本位置、作用和使用方式。

## 已完成实现

1. 新增仓库脚本 `scripts/dist-lpk.sh`。
2. 脚本会先读取 `package.yml` 中的包名，再删除仓库根目录下同包名前缀的旧 `.lpk` 文件。
3. 删除完成后，脚本会调用现有的 `desktop/scripts/build-lpk.js`，继续走当前正式 `.lpk` 打包链路。
4. 在 `package.json` 中新增 `npm run dist:lpk:script` 快捷命令。
5. 同步更新 `AGENTS.md` 与 `CLAUDE.md`，补充脚本的用途、位置和推荐使用方式。
