# REQ-20260531-001 优化创作页 Agent Prompt 的意图判断与完整编辑约束

## 基本信息

- 日期：2026-05-31
- 分类：功能优化
- 状态：已完成
- 来源：用户提出

## 需求标题

优化 AI 创作页系统 / Agent Prompt，降低普通讨论被误判为编辑的概率，并进一步强化局部编辑时的完整块输出约束。

## 背景

当前创作页已经具备规划器、编辑执行器和块级 `str_replace` 应用链路，但 Prompt 仍有两类可优化点：

1. 规划器里“无关键词时优先考虑 edit”的表达偏强，可能让普通讨论或建议类问题误入编辑链路。
2. 编辑执行 Prompt 已要求 `replace.new` 输出完整目标块，但仍缺少短示例和“无法保证完整 old/new 时不要生成操作”的明确兜底。

## 本次目标

1. 将规划器规则改为：没有明确修改动词时，只有用户明确引用块、承接上一轮编辑建议，或使用写入 / 替换 / 应用到正文等表达，才偏向 `edit`。
2. 为编辑执行 Prompt 增加极短示例，强调局部修改时 `new` 仍必须是完整目标块全文。
3. 明确目标块与上下文块边界：目标块完整处理，非目标块只用于理解上下文，不得进入 `replace/delete` 操作。
4. 增加兜底：如果无法保证 `old/new` 是完整目标块全文，返回空 `operations`。
5. 补充回归测试，锁住关键 Prompt 文案和规划器不再包含过强的“无关键词优先 edit”规则。
