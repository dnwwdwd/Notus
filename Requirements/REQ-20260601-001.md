# REQ-20260601-001 持久化工作区浏览位置并支持 AI 历史对话删除

## 基本信息

- 日期：2026-06-01
- 分类：功能优化
- 状态：已完成
- 来源：用户提出

## 需求标题

持久化侧边栏 tab 与浏览位置，补齐知识库 / 创作历史对话删除，并精简模型配置页说明文案。

## 背景

当前文件页、知识库页和创作页之间已经共享当前文件与侧边栏基础状态，但用户跨页面返回后仍可能需要重新切换侧边栏文件树 / 大纲 tab，并重新滚动到之前的阅读位置。知识库页与创作页的历史对话也只能恢复和新建，缺少整条会话删除入口。模型配置页仍保留较多说明性段落。

## 本次目标

1. 记住侧边栏文件树 / 大纲 tab、文件树与大纲各自的滚动位置。
2. 记住文件页、知识库页文档编辑区以及创作页块区的浏览位置，并按页面和文件隔离。
3. 显式来源定位、URL 行号定位和知识库来源定位优先于历史浏览位置。
4. 知识库页与创作页历史对话抽屉支持删除整条会话，删除前二次确认。
5. 删除当前会话后回到新对话空态；创作页保留当前文章块内容和未保存状态。
6. 模型配置页删除说明性段落，保留字段、占位符、按钮、测试状态和错误提示。

## 已完成实现

1. 工作区状态新增 `sidebarActiveTab` 和 `sidebarScrollByTab`，并通过统一归一化函数持久化。
2. 侧边栏文件树和大纲共用滚动容器，但按 tab 分别保存和恢复滚动位置；搜索、导入、导出、新建文件和目录会主动切回文件树并同步状态。
3. 新增浏览位置工具，按 `files:file:<id>`、`knowledge:file:<id>`、`canvas:file:<id>` 保存位置；文件 / 知识库优先按文本锚点恢复，创作页优先按 block id 和块内偏移恢复。
4. 文件页、知识库页、创作页已接入卸载保存、滚动节流保存和加载完成后恢复；显式定位不会被历史浏览位置覆盖。
5. `ConversationDrawer` 新增删除图标和二次确认；知识库页与创作页删除当前会话后回到新对话空态。
6. 后端新增 `deleteConversation(id)`，`DELETE /api/conversations/:id` 删除不存在会话时返回 404，删除成功返回 204。
7. 模型配置页已删除说明性段落、LLM 卡片解释句、默认配置说明条和弹窗自动识别说明句。
8. 已补充工作区状态、会话删除和 UI 回归测试。
