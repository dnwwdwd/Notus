# REQ-20260608-001 补齐私有 GitLab 仓库的提交范围与双远程规则

## 基本信息

- 日期：2026-06-08
- 分类：功能优化
- 状态：已完成
- 来源：用户提出

## 需求标题

为仓库增加 GitHub / 私有 GitLab 双远程协作规则，并调整忽略策略，让私有 GitLab 保留除打包产物和本地运行文件外的完整仓库内容。

## 背景

当前仓库除 GitHub 远程外，还需要同步到部署在 NAS 上的私有 GitLab。现有 `.gitignore` 额外忽略了 `AGENTS.md`、`CLAUDE.md`、`README.md`、`Requirements/`、`docs/`、`package.yml` 和懒猫相关配置文件，这会导致私有仓库无法保留完整源码、文档和项目规则。

## 本次目标

1. 在 `AGENTS.md` 与 `CLAUDE.md` 中写清楚双远程的用途和提交范围差异。
2. 明确私有 GitLab 需要纳管完整仓库内容，只排除 `.lpk`、构建输出目录和本地运行文件。
3. 调整 `.gitignore`，停止忽略项目文档与打包配置源文件。
4. 将当前仓库接入 NAS 上的私有 GitLab 远程并完成推送。

## 影响范围

- `.gitignore`
- `AGENTS.md`
- `CLAUDE.md`
- `Requirements/LEDGER.md`

## 验证计划

1. 检查 `git status --ignored --short`，确认文档与配置文件不再被忽略。
2. 检查 `git remote -v`，确认已存在私有 GitLab 远程。
3. 使用 GitLab 专用 SSH 密钥验证连接并执行推送。

## 已完成实现

1. 在协作规则中新增双远程说明，明确 GitHub 远程之外还维护 NAS 私有 GitLab 远程。
2. 明确私有 GitLab 默认提交源码、文档、需求台账、懒猫兼容配置等完整文件。
3. 明确私有 GitLab 继续排除 `.lpk`、`web-dist/`、`desktop/dist/`、`desktop/resources/notus/`、`lzc-dist/`、`.next/`、`node_modules/`、数据库、日志和本地环境变量文件。
4. 收紧 `.gitignore`，恢复项目文档与懒猫配置源文件的 Git 跟踪能力。
