# REQ-20260619-001 LLM 配置支持 OpenAI API 与 Anthropic 兼容协议

## 分类

功能优化

## 背景

当前 LLM 配置默认按 OpenAI 兼容接口保存和调用，新增或编辑配置时还要求先测试连通性。用户希望 LLM 配置同时支持 OpenAI API 与 Anthropic 两类兼容协议，并简化设置页和引导页中的 LLM 配置展示与保存流程。

## 需求

1. LLM 新增/编辑弹窗增加“兼容协议”下拉框，支持 `OpenAI API` 与 `Anthropic`。
2. 默认兼容协议为 `OpenAI API`，历史已保存配置按 `OpenAI API` 处理。
3. 新增/修改 LLM 配置不再要求先测试连通性，去除相关保存校验与提示文案。
4. 设置页和引导页的 LLM 配置列表去除卡片边框色与背景色，只显示配置名称、Provider name、模型名和 Base URL。
5. LLM 配置列表不展示“已保存密钥”、默认配置提示，也不展示兼容协议。
6. 新增/编辑弹窗去除“设为默认配置”选项；知识库页和创作页以输入框模型下拉框当前选择作为全局模型选择。

## 实现摘要

1. `llm_configs` 新增 `api_protocol` 字段，默认值为 `openai`，旧配置通过非破坏性迁移自动补齐。
2. LLM 调用层按 `api_protocol` 分流：
   - `openai` 使用 `/chat/completions`。
   - `anthropic` 使用 `/messages`，并转换 system/messages、响应内容和流式事件。
3. LLM 配置新增/编辑接口去除连接测试令牌校验。
4. 设置页与引导页复用的 LLM 配置组件改为朴素列表和协议下拉表单。
5. 知识库页和创作页的模型下拉选择会同步为全局模型选择，设置页不再提供单独默认配置入口。
6. 已同步更新 PDD、PRD、UI Guide、PROGRESS 和知识库业务流程文档。
