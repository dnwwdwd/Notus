# REQ-20260619-002 按设计稿还原知识库与创作 Agent 工作台

## 分类

用户体验优化

## 背景

用户提供 Notus-design-draft/notus-agent.html，要求将 AI 聊天页面、输入框、模型配置页面、搜索商配置页面和工具过程按设计稿还原，并与现有知识库问答和创作 Agent 业务接口对接。

## 需求

1. /knowledge 和 /canvas 整页改为设计稿风格的 Notus Agent Workspace，不再保留旧的常驻左右分栏。
2. 顶部提供模型配置和搜索配置入口，配置页在 Agent 工作台内切换。
3. 聊天区还原欢迎态、消息列表、Notus Agent 头像、工具过程、文件变更卡片、详情弹窗和撤销确认。
4. 底部输入框还原附件、联网开关、搜索服务商选择、模型选择、发送和停止按钮。
5. 知识库页继续调用 /api/chat，保留本地检索、引用来源和 SSE 流式回答。
6. 创作页继续调用 /api/agent/run，把 thinking / batch_* / assistant_meta / done 事件映射为工具过程和修改预览。
7. 搜索服务商先完成配置保存、API Key 保存状态、输入框开关和请求参数接入；本次不接真实外部搜索调用。

## 实现摘要

1. 新增共享 AgentWorkspace，统一承载知识库页和创作页的 Agent 页面、输入框、配置视图、工具过程和变更详情。
2. 新增搜索服务商配置读写接口，使用 settings 表保存 Firecrawl、Tavily、Exa、智谱的启用状态、调用模式、结果数和 API Key。
3. /api/chat 和 /api/agent/run 接受并记录 webSearchEnabled / searchProvider / attachments 元数据，当前不改变检索和创作结果。
4. 知识库页改为 Agent 工作台展示，继续消费 /api/chat SSE。
5. 创作页改为 Agent 工作台展示，当前文档内容会转换为 article blocks 后提交给 /api/agent/run；无当前文档时自动创建一篇创作草稿。
6. LLM 配置弹窗不再使用“已保存”提示文案，避免和新模型配置口径冲突。

## 验证

- npm run lint:web
- npm run build:web
- 手动检查知识库页、创作页、模型配置页、搜索配置页、底部输入框、工具过程、变更详情和撤销确认。

已由 REQ-20260620-001 修正。

本记录中的“/knowledge 和 /canvas 整页改为设计稿风格的 Notus Agent Workspace，不再保留旧的常驻左右分栏”已经废弃。最新实现以 REQ-20260620-001 为准：知识库页和创作页保留原有业务编辑/预览区，只将右侧聊天 UI、输入框、工具链和相关弹窗按设计稿还原；模型配置和搜索配置进入设置相关页面，不再从聊天顶部进入。
