# REQ-20260619-003 补充 GitLab SSH Key 与 GitHub/GitLab 远程区分规则

## 分类

功能优化

## 背景

当前仓库同时维护私有 GitLab 和公开 GitHub 两个远程。推送私有 GitLab 时需要使用专用 SSH key，且 GitHub 与 GitLab 的提交范围不同，需要在协作规则中明确。

## 需求

1. 在 AGENTS.md 中写明 GitLab 远程和 GitHub 远程的名称与地址。
2. 写明推送 GitLab 必须使用的 SSH key。
3. 写明不要用默认 SSH key 推送 GitLab。
4. 继续强调 GitLab 和 GitHub 的提交范围不同。
5. 同步更新 CLAUDE.md，保持两份协作文档一致。

## 实现摘要

1. AGENTS.md 的“Git 远程与提交范围”段落已补充远程名、远程地址和 GitLab 专用 SSH key。
2. CLAUDE.md 已同步同样内容。
3. Requirements/LEDGER.md 已新增本需求记录。

## 状态

已完成
