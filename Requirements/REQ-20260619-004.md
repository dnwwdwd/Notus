# REQ-20260619-004 创作 Agent 升级为 Agentic Loop 架构

## 分类

功能优化

## 背景

当前创作 Agent 以单轮规划和 block 级预览为主，适合轻量改写，但无法稳定覆盖整篇文档生成、跨文件整理、新建笔记、多轮工具调用和任务级回滚。

## 需求

1. 新增 Agentic Loop 会话、快照、运行日志和任务恢复能力。
2. 新增 search_knowledge、read_file、create_note、preview_patch_files、analyze_folder、check_links 六类工具。
3. 写入必须经过系统层权限校验；删除操作始终拒绝。
4. 修改已有文件必须先生成预览，用户确认后写入；新建文件记录到任务并支持回滚。
5. 创作页和知识库页继续复用现有 AgentWorkspace 输入入口，不新增独立入口。
6. 后端保留旧的块级编辑接口用于历史兼容；创作页主输入入口统一先进入 Agentic Loop 任务确认。
7. 支持 OpenAI-compatible tool_calls 与 Anthropic tool_use 两类工具调用协议。
8. 数据库字段按确认新增 canvas_operation_sets.pathes_json，用于文件级 patch 预览。

## 实现摘要

- 数据库新增 `agent_sessions`、`agent_snapshots`、`agent_run_logs`，并给 `canvas_operation_sets` 增加可空 `agent_session_id` 与 `pathes_json`。
- 新增 `agentSession / agentTools / agentLoop / agentLoopPrompt / agentSessionCleaner`，覆盖权限校验、任务快照、文件级预览、整体回滚、运行日志、连续失败/死循环/无进展检测和 waiting_confirm 超时清理。
- 新增 `/api/agent/loop/start|apply|cancel`、`/api/agent/sessions/:id`、`/api/agent/sessions/:id/rollback`；session 续跑、应用、取消、查询和回滚都要求校验 `session_token`。
- 创作页和知识库页继续复用 AgentWorkspace 输入入口；创作页主输入统一显示任务确认卡并进入 `/api/agent/loop/start`，知识库页按保守关键词规则将写作类任务分流到 Agentic Loop，普通问答继续走 `/api/chat`。
- LLM 工具调用适配 OpenAI-compatible `tool_calls` 与 Anthropic `tool_use/tool_result`，Agent system prompt 接入现有风格上下文。
- 新建和修改文件写入后触发增量索引；删除操作不作为 Agent 能力开放。
