# REQ-20260621-004 创作页 Agent 自动应用模式

- 日期：2026-06-21
- 分类：功能优化
- 状态：已完成

## 背景

创作页 Agentic Loop 已经接通，但每次发送写作任务都会先显示任务确认卡。用户希望在创作页减少一次前置确认，并通过输入框下拉框选择自动或手动模式。

## 本次调整

1. 创作页 Agent 输入框新增“自动应用 / 手动确认”下拉框，仅在创作页展示。
2. 选择值持久化到浏览器本机，跨创作页文章和对话生效，默认值为“自动应用”。
3. 自动应用模式下，发送任务后直接启动 Agentic Loop，不展示任务确认卡；当 Loop 生成文件预览后，前端自动请求 `/api/agent/loop/apply` 并在应用成功后续跑。
4. 手动确认模式保留原流程：先展示任务确认卡，再由用户手动确认预览应用。
5. 用户在自动应用请求发出前离开页面、关闭页面或切换当前对话时，不再发起自动应用；已经自动应用成功的 operation set 保留 `applied` 状态作为审批记录。

## 验证

- `npm run lint:web` 通过
- `npm run build:web` 通过
