# REQ-20260621-004 创作页 Agent 自动确认模式

- 日期：2026-06-21
- 分类：功能优化
- 状态：已完成

## 背景

创作页 Agentic Loop 已经接通，但每次发送写作任务都会先显示任务确认卡。用户希望在创作页减少前置确认，并通过输入框选择自动确认或手动确认模式。

## 本次调整

1. 创作页 Agent 输入框提供“自动 / 手动”分段确认控件，仅在创作页展示；说明文案进入 tooltip。
2. 选择值持久化到浏览器本机，跨创作页文章和对话生效，默认值为“自动确认”。
3. 自动确认模式下，发送任务后直接启动 Agentic Loop，不展示任务确认卡；当 Loop 生成文件预览后，后端自动应用所有文件，并在对话底部显示“已自动应用”的 diff 卡片。
4. 手动确认模式下也不展示前置任务确认卡；Loop 结束后在对话底部显示 diff 卡片，由用户逐文件应用或回滚。
5. 用户点击应用或回滚只调用 `/api/agent/loop/apply` 写入文件并更新 patch 状态，不触发额外 LLM 调用或 session 续跑。

## 2026-06-24 更新

- 自动应用已重命名为自动确认。
- 手动确认不再展示前置任务确认卡，改为完成后逐文件确认。
- 应用、回滚和废弃都收敛到文件级 patch 状态机。
- 选择控件已从下拉框改为双选项分段控件，自动使用闪电图标，手动使用手型图标。
