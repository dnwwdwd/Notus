# Notus 2.0 实现文档

> 作用：记录明确延期到 2.0 的能力边界，避免 1.0 页面继续暴露未交付入口。

## 当前范围

1. 本文档只记录已经在交互层被预留、但 1.0 不实际开放的功能。
2. 这些能力在 1.0 代码中应隐藏入口，而不是显示“即将接入”的占位按钮。

## 2.0 待实现能力

### 1. AI 输入框附件能力

- 支持点击 `+` 上传本地文件。
- 支持点击 `+` 上传图片。
- 支持拖拽文件或图片到输入框区域。
- 支持粘贴图片或文件作为附件。
- 支持附件预览、删除、失败重试和大小限制提示。

### 2. AI 增强来源能力

- 在输入框 `+` 菜单内增加“网络搜索”入口，并接入真实检索链路。
- 在输入框 `+` 菜单内增加更细粒度的“使用风格/指定参考”扩展操作，而不是只保留页面顶部的来源选择。

## 1.0 约束

1. 知识库页和创作页的共享 `InputBar` 不展示 `+` 按钮。
2. 不展示“网络搜索”“添加文件”“添加图片”“使用风格”等未交付菜单项。
3. 如果后续局部恢复这些入口，必须先同步更新本文档与对应产品文档。
